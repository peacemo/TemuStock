## ScUpUp WebApp系统设计（修订版V2）

---

# 第一部分：系统概述

## 1.1 系统定位

**最小可用系统**：单人操控的管理后台，用于记录多人共同投资的买卖操作，并根据数学模型自动计算每个人的持仓、成本、现金余额和盈亏。

## 1.2 核心目标

1. 准确实现数学模型的所有计算规则
2. 支持单人（管理员）录入所有成员的交易操作
3. 实时更新并展示每个人的独立账本和公共账户状态
4. 确保数据的一致性和可追溯性

# 第二部分：软件工程实现的额外约束

在数学模型的基础上，系统实现需要增加以下约束：

## 2.1 数据一致性约束

| 约束类型 | 说明 |
|----------|------|
| **事务性** | 每一笔交易（买入/卖出）必须原子性完成，同时更新所有相关个人的账本和公共账户 |
| **顺序性** | 所有操作必须严格按照时间顺序处理，不允许篡改历史记录 |
| **不可篡改性** | 历史交易记录一旦确认，不应允许修改；如需修正，必须通过反向交易冲抵 |

## 2.2 业务规则约束

| 规则 | 说明 |
|------|------|
| **资金充足检查（买入）** | 买入时，操作人的个人现金余额 + 额外入金 ≥ 计划投入金额 |
| **持股充足检查（卖出）** | 卖出时，操作人的个人持股数必须 ≥ 卖出股数 |
| **现金充足检查（提取）** | 提取现金时，操作人的个人现金余额必须 ≥ 提取金额 |
| **交易参与人非空** | 每笔交易至少有一人参与 |
| **交易数量正数** | 买入股数、卖出股数、提取金额必须 > 0 |
| **价格正数** | 交易价格必须 > 0 |
| **退出完整性** | 成员退出前必须先清仓所有持股并提取所有现金 |

## 2.3 精度约束

| 字段 | 精度要求 |
|------|----------|
| 金额 | 小数点后2位（分） |
| 股数 | 小数点后3位（根据股票类型可调整） |
| 单价 | 小数点后3位 |
| 佣金/税费 | 小数点后2位 |
| 平均成本价 | 小数点后4位（减少累积误差） |

## 2.4 审计约束

* 所有交易记录必须包含：时间戳、操作类型、参与人列表、成交价格、股数、金额、税费明细
* 系统应能随时验证守恒定律：∑持股 = 总持股，∑现金 = 总现金
* 每次交易后应记录校验和，便于追溯数据一致性问题

# 第五部分：核心业务流程

## 5.1 买入流程

1. 管理员选择交易时间、股价
2. 选择参与买入的成员，输入每人投入金额
3. 系统处理入金：
   - 对每个成员计算：入金额 = max(0, 投入金额 - 现有现金余额)
   - 若需要入金，则先增加该成员现金余额
   - 更新公共账户总现金
4. 执行买入计算引擎：
   - 计算总投入金额
   - 计算比例佣金：F_comm_raw = 总投入 × 0.0003
   - 应用最低佣金规则：F_comm_actual = max(F_comm_raw, 5)
   - 计算实际买入金额：V_buy = 总投入 - F_comm_actual
   - 按投入比例分摊实际佣金
   - 计算每人实际成交金额和股数
5. 生成交易记录和明细（包含入金信息）
6. 更新个人账本：
   - 减少个人现金（投入金额，含入金后）
   - 增加个人持股
   - 增加个人成本（实际买入金额）
   - 重新计算平均成本价
7. 更新公共账户总持股、总现金
8. 保存快照，验证守恒定律

## 5.2 卖出流程

1. 管理员选择交易时间、股价
2. 选择参与卖出的成员，输入每人卖出股数
3. 系统校验：每人持股数 ≥ 卖出股数
4. 执行卖出计算引擎：
   - 计算总成交金额、总比例佣金
   - 应用最低佣金规则，确定实际佣金
   - 计算总印花税
   - 按成交金额比例分摊实际佣金和印花税
   - 计算每人净得现金
   - 计算每人已实现盈亏
5. 生成交易记录和明细
6. 更新个人账本：
   - 增加个人现金
   - 减少个人持股
   - 减少个人成本（卖出部分）
   - 记录已实现盈亏（可选累计）
7. 更新公共账户总持股、总现金
8. 保存快照，验证守恒定律

## 5.3 新成员加入流程

1. 管理员输入新成员名称、加入时间
2. 输入注入资金金额
3. 初始化成员账本：
   - 现金余额 = 注入金额
   - 持股数 = 0
   - 累计成本 = 0
4. 更新公共账户总现金（增加注入金额）
5. **立即执行全额买入**：
   - 自动触发单人买入流程
   - 买入金额 = 注入金额
   - 按买入流程计算佣金、实际成交股数
6. 买入后成员状态：
   - 现金余额 = 0
   - 持股数 = 获得的股数
   - 累计成本 = 实际买入金额
   - 平均成本价 = 当前股价
7. 生成成员记录和初始账本快照

## 5.4 成员退出流程

1. 管理员选择要退出的成员
2. 系统检查：该成员必须已清仓所有持股
3. **步骤A：卖出全部持股**（如有持股）
   - 自动触发卖出流程
   - 卖出股数 = 成员当前全部持股
   - 按卖出流程计算费用、净得金额、已实现盈亏
   - 更新后：持股 = 0，成本 = 0，现金增加
4. **步骤B：提取全部现金**
   - 提取金额 = 成员当前全部现金余额
   - 减少个人现金至0
   - 减少公共账户总现金
5. **步骤C：标记成员退出**
   - 更新成员状态为"已退出"
   - N = N - 1
6. 生成退出记录和最终账本快照

## 5.5 现金提取流程

1. 管理员选择成员和提取金额
2. 系统校验：成员现金余额 ≥ 提取金额
3. 更新个人账本：
   - 减少个人现金余额
   - 持股和成本保持不变
4. 更新公共账户：减少总现金
5. 生成提取记录
6. 保存快照，验证守恒定律

## 5.6 除权处理流程（送股/转增）

1. 管理员输入除权日期、送转股比例 s
   - 例如：10送10，则 s = 1.0
   - 例如：10送3，则 s = 0.3
2. 系统计算总送股数：ΔQ = Q_t × s
3. 更新公共账户：总持股 = Q_t × (1 + s)
4. 更新每个成员账本：
   - 送股数 = 当前持股 × s
   - 新持股数 = 当前持股 × (1 + s)
   - 累计成本保持不变
   - 新平均成本价 = 原平均成本价 / (1 + s)
   - 现金余额保持不变
5. 生成除权记录
6. 保存快照，验证守恒定律

## 5.7 分红处理流程

1. 管理员输入分红日期、每股分红金额
2. 系统自动计算总分红金额：D = Q_t × 每股分红
3. 按当时持股比例分配至每人：d_i = q_{i,t} × 每股分红
4. 更新个人现金余额（增加分红）
5. 更新公共账户总现金（增加总分红）
6. 持股数和成本保持不变
7. 生成分红记录
8. 保存快照，验证守恒定律

# 第六部分：系统约束总结

## 6.1 功能范围（最小可用系统）

| 功能 | 是否包含 |
|------|----------|
| 成员管理（加入） | ✅ |
| 成员管理（退出） | ✅ |
| 成员管理（修改信息） | ❌ |
| 买入操作 | ✅（支持额外入金） |
| 卖出操作 | ✅ |
| 现金提取 | ✅ |
| 分红处理 | ✅ |
| 除权处理（送股/转增） | ✅ |
| 交易历史查看 | ✅ |
| 实时账本查看 | ✅ |
| 历史快照回溯 | ✅ |
| 盈亏统计 | ✅（已实现盈亏、浮动盈亏） |
| 数据修正功能 | ❌（仅通过反向交易实现） |
| 多账户支持 | ❌（仅一个公共账户） |

## 6.2 非功能约束

| 约束 | 要求 |
|------|------|
| 数据持久化 | 所有记录必须持久化存储 |
| 计算精度 | 严格遵循精度约束，避免浮点误差 |
| 操作可追溯 | 每笔操作记录操作人、操作时间 |
| 并发控制 | 单人操作，无需考虑高并发 |
| 数据备份 | 支持定期备份 |

## 6.3 边界条件处理

* **买入入金**：当成员计划投入金额 > 现有余额时，自动计算并执行入金
* **初始化**：新成员加入时，注入资金立即全部用于买入，买入后现金余额为0
* **清仓**：当成员持股数恰好等于卖出股数时，卖出后持股为0，成本为0，平均成本价未定义
* **退出**：成员退出时必须已完成清仓，持股为0，然后提取全部现金至0
* **除权**：送转股不改变成本，但降低平均成本价（除以1+送转比例）
* **极小佣金**：当总比例佣金为0时（极小交易额），直接按金额比例分摊实际佣金
* **精度处理**：当股价波动导致成交股数为非整数时，保留约定精度

# 第七部分：系统验证机制

## 7.1 实时校验

每次交易完成后，系统自动执行：

```
assert(abs(∑个人持股 - 公共总持股) < 精度阈值)
assert(abs(∑个人现金 - 公共总现金) < 精度阈值)
```

## 7.2 历史可验

系统应支持任意历史时间点的状态重建：

* 从初始状态开始
* 按时间顺序重放所有交易记录
* 应得到与快照一致的结果
