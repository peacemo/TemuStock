## ScUpUp WebApp系统设计

---

# 第一部分：系统概述

## 1.1 系统定位

**最小可用系统**：单人操控的管理后台，用于记录多人共同投资的买卖操作，并根据数学模型自动计算每个人的持仓、成本、现金余额和盈亏。

## 1.2 核心目标

1. 准确实现数学模型的所有计算规则
2. 支持单人（管理员）录入所有成员的交易操作
3. 实时更新并展示每个人的独立账本和公共账户状态
4. 确保数据的一致性和可追溯性

# 第二部分：软件工程实现的额外约束

在数学模型的基础上，系统实现需要增加以下约束：

## 2.1 数据一致性约束

| 约束类型 | 说明 |
|----------|------|
| **事务性** | 每一笔交易（买入/卖出）必须原子性完成，同时更新所有相关个人的账本和公共账户 |
| **顺序性** | 所有操作必须严格按照时间顺序处理，不允许篡改历史记录 |
| **不可篡改性** | 历史交易记录一旦确认，不应允许修改；如需修正，必须通过反向交易冲抵 |

## 2.2 业务规则约束

| 规则 | 说明 |
|------|------|
| **资金充足检查** | 买入时，操作人的个人现金余额必须 ≥ 计划投入金额 |
| **持股充足检查** | 卖出时，操作人的个人持股数必须 ≥ 卖出股数 |
| **交易参与人非空** | 每笔交易至少有一人参与 |
| **交易数量正数** | 买入股数、卖出股数必须 > 0 |
| **价格正数** | 交易价格必须 > 0 |

## 2.3 精度约束

| 字段 | 精度要求 |
|------|----------|
| 金额 | 小数点后2位（分） |
| 股数 | 小数点后3位（根据股票类型可调整） |
| 单价 | 小数点后3位 |
| 佣金/税费 | 小数点后2位 |
| 平均成本价 | 小数点后4位（减少累积误差） |

## 2.4 审计约束

* 所有交易记录必须包含：时间戳、操作类型、参与人列表、成交价格、股数、金额、税费明细
* 系统应能随时验证守恒定律：∑持股 = 总持股，∑现金 = 总现金
* 每次交易后应记录校验和，便于追溯数据一致性问题

# 第五部分：核心业务流程

## 5.1 买入流程

1. 管理员选择交易时间、股价
2. 选择参与买入的成员，输入每人投入金额
3. 系统校验：每人现金余额 ≥ 投入金额
4. 执行买入计算引擎：
   - 计算总投入、总比例佣金
   - 应用最低佣金规则，确定实际佣金
   - 按投入比例分摊实际佣金
   - 计算每人实际成交金额和股数
5. 生成交易记录和明细
6. 更新个人账本：
   - 减少个人现金
   - 增加个人持股
   - 增加个人成本
   - 重新计算平均成本价
7. 更新公共账户总持股、总现金
8. 保存快照，验证守恒定律

## 5.2 卖出流程

1. 管理员选择交易时间、股价
2. 选择参与卖出的成员，输入每人卖出股数
3. 系统校验：每人持股数 ≥ 卖出股数
4. 执行卖出计算引擎：
   - 计算总成交金额、总比例佣金
   - 应用最低佣金规则，确定实际佣金
   - 计算总印花税
   - 按成交金额比例分摊实际佣金和印花税
   - 计算每人净得现金
   - 计算每人已实现盈亏
5. 生成交易记录和明细
6. 更新个人账本：
   - 增加个人现金
   - 减少个人持股
   - 减少个人成本（卖出部分）
   - 记录已实现盈亏（可选累计）
7. 更新公共账户总持股、总现金
8. 保存快照，验证守恒定律

## 5.3 新成员加入流程

1. 管理员输入新成员名称、加入时间
2. 输入注入资金金额
3. 如立即买入，输入买入金额
4. 执行注资和买入逻辑（买入部分按单人买入处理）
5. 生成成员记录和初始账本快照

## 5.4 分红处理流程

1. 管理员输入分红日期、每股分红金额
2. 系统自动计算总分红金额
3. 按当时持股比例分配至每人
4. 更新个人现金余额
5. 更新公共账户总现金
6. 生成分红记录

# 第六部分：系统约束总结

## 6.1 功能范围（最小可用系统）

| 功能 | 是否包含 |
|------|----------|
| 成员管理（增） | ✅ |
| 成员管理（删/改） | ❌（仅支持加入，不支持删除） |
| 买入操作 | ✅ |
| 卖出操作 | ✅ |
| 分红处理 | ✅ |
| 交易历史查看 | ✅ |
| 实时账本查看 | ✅ |
| 历史快照回溯 | ✅ |
| 数据修正功能 | ❌（仅通过反向交易实现） |
| 多账户支持 | ❌（仅一个公共账户） |

## 6.2 非功能约束

| 约束 | 要求 |
|------|------|
| 数据持久化 | 所有记录必须持久化存储 |
| 计算精度 | 严格遵循精度约束，避免浮点误差 |
| 操作可追溯 | 每笔操作记录操作人、操作时间 |
| 并发控制 | 单人操作，无需考虑高并发 |
| 数据备份 | 支持定期备份 |

## 6.3 边界条件处理

* 当成员现金余额恰好等于投入金额时，买入后现金为0
* 当成员持股数恰好等于卖出股数时，卖出后持股为0，成本为0，平均成本价未定义
* 当总比例佣金为0时（极小交易额），直接按金额比例分摊实际佣金
* 当股价波动导致成交股数为非整数时，保留约定精度

# 第七部分：系统验证机制

## 7.1 实时校验

每次交易完成后，系统自动执行：

```
assert(abs(∑个人持股 - 公共总持股) < 精度阈值)
assert(abs(∑个人现金 - 公共总现金) < 精度阈值)
```

## 7.2 历史可验

系统应支持任意历史时间点的状态重建：

* 从初始状态开始
* 按时间顺序重放所有交易记录
* 应得到与快照一致的结果
